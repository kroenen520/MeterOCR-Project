## 一、准备工作

### 1.1 确保项目可运行

在打包前，先确保Python程序可以正常运行：

```bash
# 进入项目目录
cd MeterOCR-Project

# 安装依赖
pip install -r requirements.txt

# 测试运行
python main.py
```

如果能正常运行并显示菜单，说明程序没问题。

### 1.2 项目结构检查

确保项目结构如下：

```
MeterOCR-Project/
├── main.py              ✓ 主程序
├── meter_ocr.spec       ✓ 打包配置文件
├── requirements.txt     ✓ 依赖列表
├── README.md            ✓ 项目说明
├── LICENSE              ✓ 开源协议
└── .gitignore           ✓ Git忽略文件
```

## 二、安装打包工具

### 2.1 安装 PyInstaller

```bash
pip install pyinstaller
```

验证安装：
```bash
pyinstaller --version
```

### 2.2 安装UPX（可选，用于压缩）

UPX可以减小EXE文件体积。

**Windows用户：**
1. 下载UPX：https://github.com/upx/upx/releases
2. 解压后将 `upx.exe` 放到系统PATH目录，或项目根目录

**验证UPX：**
```bash
upx --version
```

## 三、配置打包参数

### 3.1 使用提供的spec文件

项目已包含 `meter_ocr.spec` 配置文件，主要配置说明：

```python
# 关键配置项

name='MeterOCR'          # EXE文件名
console=True             # 显示控制台（方便看进度）
upx=True                 # 使用UPX压缩
icon=None                # 可设置图标：'icon.ico'
```

### 3.2 自定义配置（可选）

#### 添加程序图标

1. 准备图标文件 `icon.ico`（放在项目根目录）
2. 修改 `meter_ocr.spec`：

```python
exe = EXE(
    ...
    icon='icon.ico',     # 添加这一行
)
```

#### 设置为窗口模式（不显示控制台）

如果不需要看命令行输出：

```python
exe = EXE(
    ...
    console=False,       # 改为False
)
```

> ⚠️ 注意：本项目建议保持 `console=True`，因为OCR初始化需要时间，控制台可以显示进度。

## 四、执行打包

### 4.1 执行打包命令

在项目根目录执行：

```bash
pyinstaller meter_ocr.spec
```

### 4.2 等待打包完成

打包过程可能需要 **5-20分钟**，取决于电脑性能。

输出示例：
```
INFO: PyInstaller: 5.13.0
INFO: Python: 3.10.0
...
INFO: Building EXE because EXE-00.toc is non existent
INFO: Building EXE from EXE-00.toc
INFO: Copying icons from ['icon.ico']
INFO: Writing RT_ICON 1 resource with 1128 bytes
INFO: Appending archive to EXE dist\MeterOCR.exe
INFO: Building EXE completed successfully.
```

### 4.3 打包输出

打包完成后，生成以下目录：

```
MeterOCR-Project/
├── build/               # 打包临时文件（可删除）
│   └── meter_ocr/
├── dist/                # 打包输出目录 ⭐
│   └── MeterOCR.exe     # 最终EXE文件 ⭐
└── ...
```

## 六、分发EXE

### 6.1 整理分发包

创建分发文件夹：

```
MeterOCR-v1.0.0/
├── MeterOCR.exe          # 主程序
├── 使用说明.txt          # 简要使用说明
└── 示例数据/             # 可选：示例照片
    └── 示例站点/
        └── 示例照片.jpg
```

### 6.2 使用说明模板

创建 `使用说明.txt`：

```
电表OCR批量识别工具 v1.0.0
========================

【使用方法】
1. 在程序所在目录创建文件夹
2. 将电表照片放入文件夹中
3. 双击运行 MeterOCR.exe
4. 等待识别完成，自动生成Excel表格

【输出文件】
- 格式：YYYYMMDDHHMMSS.xlsx（如 20260104152119.xlsx）
- 位置：程序所在目录

【支持格式】
- 图片：jpg, jpeg, png, bmp, gif, tiff, webp
- 站址编码：440开头，18位数字
- 电流值：0.1-100A范围

【注意事项】
- 首次运行会自动下载OCR模型（约100MB），请保持网络畅通
- 照片尽量清晰，文字端正
- 每个文件夹会被视为一个站点

【系统要求】
- Windows 10/11
- 4GB+ 内存
- 2GB+ 硬盘空间
```

### 6.3 压缩分发

将分发文件夹压缩为ZIP：

```bash
# 使用7-Zip或WinRAR
7z a MeterOCR-v1.0.0.zip MeterOCR-v1.0.0/
```

## 七、常见问题

### Q1: 打包失败，提示找不到模块

**解决**：在 `meter_ocr.spec` 的 `hiddenimports` 中添加缺失的模块：

```python
hiddenimports=[
    'paddle',
    'paddleocr',
    # 添加缺失的模块
    '缺失的模块名',
],
```

### Q2: EXE文件太大（几百MB）

**原因**：PaddleOCR依赖较多

**优化**：
1. 使用UPX压缩（已配置）
2. 使用 `--onefile` 模式（但启动更慢）
3. 分发时压缩ZIP

### Q3: EXE运行时报错"找不到模型"

**解决**：首次运行需要联网下载模型。或者手动下载模型放到正确位置。

### Q4: 杀毒软件报毒

**原因**：PyInstaller打包的程序有时会被误报

**解决**：
1. 将EXE添加到杀毒软件白名单
2. 使用代码签名证书（商业方案）
3. 向用户说明是误报

### Q5: 在其他电脑无法运行

**原因**：缺少运行库

**解决**：
1. 确保目标电脑是Windows 10/11
2. 安装 Visual C++ Redistributable
   - 下载：https://aka.ms/vs/17/release/vc_redist.x64.exe

### Q6: 打包时内存不足

**解决**：
1. 关闭其他程序
2. 增加虚拟内存
3. 使用 `--clean` 参数清理缓存：
   ```bash
   pyinstaller --clean meter_ocr.spec
   ```

## 八、高级配置

### 8.1 单文件模式（不推荐）

如果想打包成单个EXE文件（无外部依赖）：

```bash
pyinstaller --onefile --specpath . main.py
```

> ⚠️ 缺点：启动速度慢，因为每次都要解压

## 十、GitHub发布

### 10.1 创建Release

1. 在GitHub仓库点击 "Releases"
2. 点击 "Create a new release"
3. 填写版本号（如 v1.0.0）
4. 上传打包好的ZIP文件
5. 发布

### 10.2 自动打包（GitHub Actions）

创建 `.github/workflows/build.yml`：

```yaml
name: Build EXE

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    runs-on: windows-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.10'
    
    - name: Install dependencies
      run: |
        pip install -r requirements.txt
        pip install pyinstaller
    
    - name: Build EXE
      run: pyinstaller meter_ocr.spec
    
    - name: Upload artifact
      uses: actions/upload-artifact@v3
      with:
        name: MeterOCR
        path: dist/MeterOCR.exe
```
